通过双链表实现线程安全的LruCache

发表于 2019-04-19 | 更新于 2019-04-20 | 分类于 java

LruCache实际上是模仿android的LruCache来实现的，但因为android的LruCache是通过synchronized和LinkHashMap实现的。而我自定义的是双向链表、HashMap和ReadWriteLock来实现。

  • 链表：插入到合适位置
  • HashMap：直接通过key获取节点
  • ReadWriteLock：当增加、移动、删除的时候用写锁，而查阅的时候用读锁。

双向链表

模仿LinkHashMap的双向链表实现方式来实现LruCache的双向链表。
其中元素的实现类如下：
private static class Entry<K, V> {
Entry<K, V> before;//上一个节点
Entry<K, V> after;//下一个节点
final V value;//保存的内容
final K key;//键

    Entry(K key, V value) {
        this.key = key;
        this.value = value;
    }

    public V getValue() {
        return value;
    }

    public K getKey() {
        return key;
    }
}

双向链表的实现：

private Entry<K, V> head;//链表的头部
private Entry<K, V> tail;//链表的尾部

有新的节点都是从尾部开始添加，在首次插入节点时，将头尾都指向p（head = last = p），实现代码如下：

private void linkNodeLast(Entry<K, V> p) {
    // 添加元素之前双向链表尾部节点
    Entry<K, V> last = tail;
    // tail 指向新添加的节点
    tail = p;
    //如果之前 tail 指向 null 那么集合为空新添加的节点 head = tail = p
    if (last == null)
        head = p;
    else {
        // 否则将新节点的 before 引用指向之前当前链表尾部
        p.before = last;
        // 当前链表尾部节点的 after 指向新节点
        last.after = p;
    }
}

删除节点时，实现代码如下：

private void afterNodeRemoval(Entry<K, V> e) {
    Entry<K, V> p = e, b = p.before, a = p.after;
    // 将 p 节点的前后指针引用置为 null 便于内存释放
    p.before = p.after = null;
    // p.before 为 null，表明 p 是头节点
    if (b == null)
        head = a;
    else//否则将 p 的前驱节点连接到 p 的后驱节点
        b.after = a;
    // a 为 null，表明 p 是尾节点
    if (a == null)
        tail = b;
    else //否则将 a 的前驱节点连接到 b
        a.before = b;
}

被访问的节点都会移动到链表最后

private void afterNodeAccess(Entry<K, V> e) { 
    // move node to last
    Entry<K, V> last;
    if ((last = tail) != e) {
        Entry<K, V> p = e, b = p.before, a = p.after;
        //访问节点的后驱置为 null
        p.after = null;
        //如访问节点的前驱为 null 则说明 p = head
        if (b == null)
            head = a;
        else
            b.after = a;
        //如果 p 不为尾节点 那么将 a 的前驱设置为 b
        if (a != null)
            a.before = b;
        else
            last = b;

        if (last == null)
            head = p;
        else {
            p.before = last;
            last.after = p;
        }
        tail = p;// 将 p 接在双向链表的最后
    }
}

这些就是双向链表的基础实现

读写锁（ReadWriteLock）

可以多个线程同时获取读取锁，只有一个线程能获取写入锁，而且在进行写操作时，其他线程都不能进行读和写的操作。

  • 可重入锁
    允许writer和reader获取重新获取读取锁和写入锁。writer可以获取读取锁，但反过来不成立。

  • 锁降级
    重入允许从写入锁降级为读取锁，其实现方式是：先获取写入锁，然后获取读取锁，最后释放写入锁。但是，从读取锁升级到写入锁是不可能的。就拿LruCache的检测函数作为示例代码，如下

总结

参照android的LruCache实现的自定义LrcCache基本实现原理就是用

  • 双向链表：排序和指向性的插入
  • HashMap：通过Key获取节点
  • ReadWriteLock：实现线程安全
